# Voxel Vengeance

Voxel Vengeance is a top-down low-poly multiplayer shooter set in a desert environment.  
The project focuses on real-time multiplayer gameplay with a server-authoritative architecture and client-side prediction for smooth movement and responsiveness.

---

## 🎮 Core Features

- Real-time multiplayer top-down shooting gameplay
- Server-authoritative architecture
- Client-side prediction and server reconciliation
- Damage system with hit registration
- Kill / death notifications
- Hit markers and damage numbers
- Low-poly desert environment
- All 3D models are custom-made

---

## 🔫 Weapons

The game features 4 different weapons, each with unique firing behavior:

- Fast single-shot weapon
- High fire-rate spray weapon
- Multi-projectile weapon
- High-damage slow weapon

Each weapon has different shooting patterns, recoil behavior, and projectile logic.

---

## 🧠 Networking & Architecture

The game uses a server-authoritative multiplayer model to ensure fair gameplay and consistency.

Key systems include:

- Client-side prediction for responsive movement
- Server reconciliation for correcting client state
- Server-controlled hit validation and damage handling
- Authoritative game state synchronization

---

## ⚙️ Game Systems

- Main menu (prototype stage)
- Shooting system fully implemented
- Damage and health system
- Basic gameplay loop (FFA mode)

---

## 🧪 Notes

This project is a prototype developed to explore multiplayer game architecture, real-time networking, and gameplay systems.

Some features (such as ammo management and full polish of all systems) are not fully implemented.

---

## 🛠 Tech Focus

- Multiplayer networking (client-server architecture)
- Real-time synchronization
- Game state prediction and reconciliation
- Game loop design
- Custom 3D assets integration

---

## 🚧 Status

Voxel Vengeance is a playable prototype but not a finished game.  
The focus of the project was learning multiplayer architecture and real-time gameplay systems rather than full production release.
